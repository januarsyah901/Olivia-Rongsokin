# Plan 01-02: User Authentication API (JWT, Bcrypt)

## Goal
Implement the registration and login API endpoints with JWT cookies.

## Tasks

<task>
  <action>Create `server/src/controllers/auth.ts` containing `register` and `login` handlers using `bcrypt` for hashing and `jsonwebtoken` for HTTP-only cookies.</action>
  <read_first>server/prisma/schema.prisma</read_first>
  <acceptance_criteria>
    - `auth.ts` exports `register` and `login` functions.
    - Passwords are hashed before saving to DB.
    - JWT is sent via `res.cookie` with `httpOnly: true`.
  </acceptance_criteria>
</task>

<task>
  <action>Create `server/src/routes/auth.ts` and mount it in `server/src/index.ts`.</action>
  <read_first>server/src/index.ts</read_first>
  <acceptance_criteria>
    - `server/src/routes/auth.ts` exposes POST `/api/auth/register` and POST `/api/auth/login`.
    - Endpoints are mounted in `index.ts` at `/api/auth`.
  </acceptance_criteria>
</task>

## Verification
- Using curl or Postman, send a POST request to `/api/auth/register` and verify a 201 status with an HTTP-only cookie.
